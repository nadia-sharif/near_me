# near_me





This app is the same as Accessibility Neighbourhood but it uses Mapbox Streets v8 vector tileset's 'poi_label' layer for finding Amenities instead of Open Street Maps POI layers. The amenities found are better categorized than the OSM data.

This is the Version 2 of the "Accessibility Neighbourhood" and has only two modes "Walking" and "Cycling". There is a known bug in the app as follows: When user changes the Mode, like from Walking to Cycling, the map boundaries reset to fit the bigger isochrones generated by this mode, however, at the same time the logic for finding POIs within the isochrones is fired. Since the map is resetting bounds at the same time, only a few POIs are returned. I tried fixing this issue my clearing the map whenever the user changes mode, but the bug remains unfixed.
